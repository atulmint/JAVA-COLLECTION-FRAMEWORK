## ğŸ“˜ **1. What Is an ArrayList?**

* `ArrayList` is a **resizable array implementation** of the `List` interface in Java.
* It belongs to the `java.util` package.
* Unlike regular arrays, it can **grow and shrink dynamically** as elements are added or removed.
* Internally, it uses a **dynamic array (`Object[]`)** to store elements.

---

## âš™ï¸ **2. Key Characteristics**

| Feature                 | Description                                         |
| ----------------------- | --------------------------------------------------- |
| **Data Structure Used** | Dynamic array                                       |
| **Implements**          | `List`, `RandomAccess`, `Cloneable`, `Serializable` |
| **Ordering**            | Maintains **insertion order**                       |
| **Duplicates**          | Allowed                                             |
| **Null Values**         | Allowed                                             |
| **Thread Safety**       | Not synchronized (not thread-safe)                  |
| **Resizing Mechanism**  | Grows by 50% when capacity exceeds                  |

---

## ğŸ§  **3. Internal Working**

### ğŸ”¹ **Initial Capacity**

* When an `ArrayList` is created, it has a **default capacity** (usually 10).
* Example:

  ```java
  ArrayList<Integer> list = new ArrayList<>();
  ```

  This initializes an internal array of size 10.

---

### ğŸ”¹ **Resizing Process**

When you add more elements than the current capacity:

1. A **new larger array** is created (1.5Ã— the old size).
2. Existing elements are **copied** to the new array.
3. The old array is discarded.

This resizing operation is **time-consuming**, so itâ€™s best to set an initial capacity if you know the expected size.

Example:

```java
ArrayList<Integer> numbers = new ArrayList<>(100);
```

## ğŸ”§ **4. Commonly Used Methods**

| Method                      | Description                                    |
| --------------------------- | ---------------------------------------------- |
| `add(E e)`                  | Adds element to the end of the list.           |
| `add(int index, E element)` | Inserts element at specified position.         |
| `get(int index)`            | Returns element at given index.                |
| `set(int index, E element)` | Replaces element at given index.               |
| `remove(int index)`         | Removes element at given index.                |
| `remove(Object o)`          | Removes first occurrence of specified element. |
| `size()`                    | Returns number of elements in the list.        |
| `isEmpty()`                 | Checks if list is empty.                       |
| `clear()`                   | Removes all elements.                          |
| `contains(Object o)`        | Checks if list contains an element.            |
| `indexOf(Object o)`         | Returns index of first occurrence.             |
| `lastIndexOf(Object o)`     | Returns index of last occurrence.              |

---

## ğŸ§® **5. Time Complexity**

| Operation             | Average Case | Description                                  |
| --------------------- | ------------ | -------------------------------------------- |
| `add(E e)`            | O(1)*        | Amortized constant time; may trigger resize. |
| `add(index, E e)`     | O(n)         | Needs element shifting.                      |
| `get(int index)`      | O(1)         | Direct index access (like arrays).           |
| `set(int index, E e)` | O(1)         | Direct index replacement.                    |
| `remove(int index)`   | O(n)         | Requires shifting elements.                  |
| `contains(Object o)`  | O(n)         | Needs linear search.                         |

> âš ï¸ *Amortized O(1)*: Usually constant time, but occasionally O(n) when resizing occurs.
